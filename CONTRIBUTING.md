# Como Contribuir

Ficamos felizes por você querer contribuir com o projeto! Existem várias maneiras de nos ajudar, e agradecemos por cada uma delas.

## Reportando Bugs

Se você encontrar um bug, por favor, abra uma "issue" em nosso repositório no GitHub. Ao reportar um bug, inclua o máximo de detalhes possível:

- Uma descrição clara e concisa do que é o bug.
- Passos para reproduzir o comportamento.
- O que você esperava que acontecesse.
- O que realmente aconteceu (incluindo logs, screenshots, etc.).
- Informações sobre o seu ambiente (versão do Java, sistema operacional, etc.).

## Sugerindo Melhorias

Se você tem uma ideia para uma nova funcionalidade ou uma melhoria para uma existente, abra uma "issue" com a tag "enhancement". Descreva sua sugestão em detalhes, explicando por que ela seria útil para o projeto.

## Submetendo Pull Requests

Para contribuir com código ou documentação, siga os passos abaixo:

1. **Faça um Fork do Repositório:**
    Clique no botão "Fork" no canto superior direito da página do repositório para criar uma cópia do projeto na sua conta do GitHub.

2. **Clone o seu Fork:**

    ```bash
    git clone https://github.com/PedroZef/Industries-Fharmacia-Abstergo_AWS.git
    cd Industries-Fharmacia-Abstergo_AWS
    ```

3. **Crie uma Nova Branch:**
    Crie uma branch para a sua contribuição. Use um nome descritivo.

    ```bash
    git checkout -b minha-nova-feature
    ```

4. **Faça suas Alterações:**
    Escreva seu código, garantindo que ele siga o estilo e as convenções do projeto. Adicione testes para suas alterações, se aplicável.

5. **Faça o Commit das suas Alterações:**
    Use uma mensagem de commit clara e descritiva.

    ```bash
    git commit -am "Adiciona nova feature incrível"
    ```

6. **Envie para o seu Fork:**

    ```bash
    git push origin minha-nova-feature
    ```

7. **Abra um Pull Request:**
    Vá para o repositório original e abra um Pull Request. Preencha o template do PR com as informações solicitadas.

Agradecemos sua contribuição!
